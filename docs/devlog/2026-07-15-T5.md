# T5 — Жеребьёвка (2026-07-15)

## Что сделал

**Backend:**
- Enum `MatchStatus` (pending/done) и сущность **`BracketMatch`** (назвал так, потому что `Match` — зарезервированное слово в PHP 8 и MySQL). Поля: tournament, tableNumber, round (1-based), slot (0-based), player1/player2/winner (nullable User), status, playedAt. Хелперы `setWinner()` (ставит done+playedAt), `isReady()`.
- `BracketMatchRepository`: countByTournament, findByTournamentOrdered, findOneBySlot (пригодится в T6).
- `DrawService::draw()`:
  - гварды: матчи уже есть → 409 «уже проведена»; статус не `checkin` → 409 «сначала закройте чекин»; пустой ростер → 422;
  - берёт `registered`-ростер, `shuffle()`;
  - **сплит по столам: стол 1 до 16, остаток на стол 2** (не пополам); проставляет `entry.tableNumber`;
  - для каждого стола строит сетку: `bracketSize` = ближайшая степень двойки ≥ числа игроков; первый тур = реальные пары + автопроходы (bye); байи сразу помечаются сыгранными и их победители продвигаются во второй тур; статус турнира → `drawn`.
- Эндпоинт `POST /api/admin/tournaments/{id}/draw` (ROLE_ADMIN) → 201 `{status, tables:{table1,table2}}`.
- Миграция `bracket_match`.
- Тесты `DrawServiceTest` (+5, всего **27**): 32→16/16 (полные сетки, 30 матчей, без байев); 20→16/4; 12→12/0 (4 автопрохода, продвинуты во 2-й тур, нет пустых матчей); гварды (статус, повторная жеребьёвка). Все зелёные.

**Frontend:**
- На `CheckinPage` после закрытия чекина (статус `checkin`) — кнопка **«Провести жеребьёвку»**; после (`drawn`) — «Жеребьёвка проведена» + ссылка «Открыть сетку» (страница сетки — T7). API `drawTournament`.

## Почему так

- **`BracketMatch`, а не `Match`** — `match` в PHP 8 нельзя использовать как имя класса (ключевое слово выражения match), а `match` в MySQL — зарезервированное слово таблицы. Сразу заложил безопасное имя.
- **Байи размещаю «реальные пары, потом автопроходы»** и доказано, что автопроходов всегда `< bracketSize/2` (т.к. bracketSize — наименьшая степень двойки ≥ N), поэтому bye-vs-bye в первом туре невозможен — у каждого автопрохода есть живой игрок или он один в паре. Победители байев сразу проброшены во второй тур, чтобы сетка была консистентной с самого начала.
- **Связь туров формулой** `(round r, slot s) → (round r+1, slot s>>1)`, player1 если slot чётный, иначе player2. Тот же метод переиспользует `AdvanceService` в T6.
- **Сплит «стол 1 до 16, остаток на стол 2»** — ровно как просил пользователь (не делим пополам): 20 → 16/4, а не 10/10.
- **Draw требует статус `checkin`** (чекин закрыт) — гарантия, что ростер финальный (но-шоу убраны, очередь добрана).

## Как проверить

Проверено вживую (curl, турнир id=1): 5 walk-in игроков → `draw` до close-checkin даёт 409; после close-checkin `draw` → 201 `{table1:5, table2:0}`; в БД сетка на 8 (тур1=4 [1 реальный + 3 bye], тур2=2, финал=1). Dev прибран.

Через браузер: админ → «Турниры» → «Экран чекина» → «Закрыть чекин» → «Провести жеребьёвку».

Тесты: `make test` → `OK (27 tests)`.
