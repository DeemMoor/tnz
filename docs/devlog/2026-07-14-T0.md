# T0 — Каркас проекта (2026-07-14)

## Что сделал

- Развернул стек как у jegebce.online: **Symfony 7.4 + React/Vite + MySQL 8**, локально в Docker.
- Docker-окружение (`compose.yaml`): контейнеры `php` (8.5-fpm), `nginx` (docroot `backend/public`, сайт на **http://localhost:8090**), `db` (MySQL 8, хостовый порт **3308**), `node` (Vite на **5175**). Порты сдвинуты, чтобы не конфликтовать с другими локальными проектами.
- `Makefile` с короткими командами (`make up`, `make sh`, `make composer`, `make console`, `make test`, `make front-build`, `make deploy`).
- Backend: Symfony 7.4.14 skeleton в `backend/`. Поставил бандлы: Doctrine ORM + migrations, Security, Serializer, Validator, Mailer, HttpClient, Nelmio CORS, Nelmio ApiDoc, EasyAdmin, Maker, Fixtures.
- Коннект к MySQL (креды в `backend/.env.local`, в git не идут). Эндпоинт `GET /api/ping` → `{"status":"ok","db":true}`.
- Frontend: React 19 + TS + Vite в `frontend/`, роутинг через `react-router-dom`. Страница-заглушка дёргает `/api/ping`. Vite настроен: host, порт 5175, proxy `/api` → nginx.
- EasyAdmin на `/admin` (пустой дашборд).
- Документы: `CLAUDE.md` (правила проекта), `docs/ROADMAP.md` (план T0–T8), `.gitignore`, `scripts/deploy-remote.sh`.

## Почему так

- **Стек 1:1 с jegebce** — по прямой просьбе: тот же хостинг (BeGet shared), те же грабли уже пройдены, деплой по SSH переиспользуется.
- **Порт сайта 8090**, а не 8081 — 8081 на машине уже занят другим сервисом (nginx не поднимался с «port already allocated»). 8090 свободен.
- **node working_dir = `/app`, а не `/app/frontend`** — иначе Docker при старте создаёт `frontend/` от root ещё до переключения на пользователя 1000, и потом туда не записать (уже наступили: пришлось удалять root-овую папку). Внутрь `frontend` заходим в `command`.
- **Vite proxy `/api` → nginx** — чтобы локально фронт и бэк жили на разных портах, но без CORS. На проде оба на одном домене, там прокси не нужен.
- **EasyAdmin: контроль доступа пока не вешал** — сущности `User` и security-конфига ещё нет (это T1). Сейчас `/admin` открыт; закрывается в T1.
- **Секреты в `.env.local`** (gitignored), в `.env` только формат-плейсхолдеры — как требует раздел «Безопасность» в CLAUDE.md.

## Как проверить

1. `cd /home/dmitriy/projects/nz && make up` — поднимутся 4 контейнера.
2. Бэкенд/БД: открыть **http://localhost:8090/api/ping** → должно быть `{"status":"ok","db":true}`.
3. Фронт: открыть **http://localhost:5175/** → страница «Настольный теннис — турниры», внизу зелёным «API: ok, БД: ok» (фронт сам сходил на API через прокси).
4. Админка: **http://localhost:8090/admin** → страница EasyAdmin (пока пустая).
