# T3 — Турниры и регистрация (2026-07-15)

## Что сделал

**Backend:**
- Enum-ы `TournamentStatus` (draft/registration/checkin/drawn/in_progress/finished) и `EntryStatus` (registered/waitlisted/cancelled/dropped).
- Сущности `Tournament` (name, `date` = воскресенье, status, `CAPACITY=32` const) и `TournamentEntry` (status, `registeredAt` для порядка, checkedIn/checkedInAt, tableNumber, eliminated; unique tournament+user). Миграция.
- `TournamentSchedule` — считает окна **из даты**: регистрация с чт 16:00 (`date -3 дня`), чекин вс 14:00–14:15. `isRegistrationOpen`/`isCheckinOpen`. Время берёт из `ClockInterface` (в тестах подменяется).
- `RegistrationService` — вся логика записи:
  - гейты: окно регистрации открыто; не чемпион; RTTF ≤ 250; не дубль;
  - первые 32 → `registered`, дальше → `waitlisted` (порядок по `registeredAt`);
  - `unregister` → `cancelled` + **автоподъём первого из очереди**, если освободилось место в основе;
  - `promoteFirstWaitlisted`.
- `TournamentController`: `GET /api/tournaments` (список, публично), `GET /api/tournaments/{id}`, `POST .../register`, `DELETE .../registration` (оба под `ROLE_USER`). View отдаёт расписание, счётчики основы/очереди и мой статус (позиция в очереди).
- EasyAdmin: `TournamentCrudController` (создать турнир, дата, статус), `UserCrudController` (просмотр игроков), пункты меню в дашборде.
- Установил `symfony/clock` — тестируемое время.
- Тесты (+8, всего **18**): `RegistrationServiceTest` (основа/очередь на 33 игроках, автоподъём) и `TournamentApiTest` (гейты: открыто/закрыто/дубль/чемпион/RTTF>250/снятие). Все зелёные.

**Frontend:**
- Тип `Tournament`/`MyEntry`, слой `api/tournaments.ts`.
- `TournamentsPage` (список) + `TournamentCard` (дата, счётчики, мой статус-бейдж, кнопка «Участвовать»/«Снять с регистрации», подсказка «регистрация откроется …»). Ссылка «Турниры» на главной.

## Почему так

- **Логику записи вынес в `RegistrationService`**, а не в контроллер — тонкий контроллер + тестируемое ядро (правило CLAUDE.md). Гейты и очередь — критичная логика, покрыта тестами.
- **Время через `ClockInterface`, а не `new DateTimeImmutable()`** — иначе временную логику (окна чт–вс) не протестировать. В тестах `MockClock` зафиксирован на 2026-07-17 12:00, нужное окно подбираю датой турнира.
- **Порядок в основе и очереди — по `registeredAt`**, а не по id — семантически это «время записи»; при реактивации после снятия человек встаёт в конец.
- **Баг, который поймал тестом:** считал места из БД до `flush()` отмены — очередь не подтягивалась. Теперь `unregister` флашит отмену перед подсчётом. Так же `view()` показывает `me` только для активных статусов (после снятия — `null`).
- **Гейт RTTF>250** — строго больше (≤250 пускаем), причина возвращается текстом в 403.

## Как проверить

Через браузер (http://localhost:5175/ → «Турниры»): виден «Воскресный турнир» на 2026-07-19, «Основа 0/32». Регистрация закрыта (сегодня раньше чт 16:00) — показывает «Регистрация откроется чт, 16 июля, 16:00».

Через API (проверено): на турнире с открытым окном `POST /api/tournaments/{id}/register` → 201 (`me.status=registered`), `DELETE .../registration` → `me=null`. Гейты и очередь — в тестах.

Тесты: `make test` → `OK (18 tests)`. Админка: http://localhost:8090/admin → «Турниры» (создание/правка).
