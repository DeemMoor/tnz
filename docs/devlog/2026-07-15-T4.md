# T4 — Чекин и обработка очереди (2026-07-15)

## Что сделал

**Backend:**
- `CheckinService`:
  - `checkIn(t, user, byAdmin, now)` — отметить пришедшим. Сам игрок — только в окне вс 14:00–14:15 и только из основы (`registered`); админ (`byAdmin`) — в любой момент.
  - `walkIn(t, phone, name)` — найти/создать игрока по телефону (нормализация, авто-пароль), поставить на турнир в обход окна регистрации и сразу чекинить (в основу, если есть место).
  - `closeCheckin(t)` — незачекиненные из основы → `dropped`; добор из очереди по порядку (первому — независимо от чекина); статус турнира → `checkin`. Возвращает `{dropped, promoted}`.
- `RegistrationService::register(..., ignoreSchedule)` — флаг для walk-in (обойти проверку окна).
- Эндпоинты:
  - `POST /api/tournaments/{id}/checkin` — самоотметка игрока (ROLE_USER).
  - `AdminTournamentController` (ROLE_ADMIN): `GET /roster`, `POST /checkin/{userId}`, `POST /walk-in`, `POST /close-checkin`.
- `security.yaml`: `^/api/admin` → ROLE_ADMIN (защита в глубину поверх `#[IsGranted]`).
- Тесты `CheckinServiceTest` (+4, всего **22**): самоотметка в окне / вне окна (ошибка), walk-in создаёт юзера и чекинит, закрытие чекина сбрасывает 2 но-шоу и добирает 2 из очереди до 32. Все зелёные.

**Frontend:**
- В `TournamentCard`: кнопка «Я на месте — отметиться» (когда я в основе, не отмечен и окно чекина открыто) + ссылка «Экран чекина (админ)» для ROLE_ADMIN.
- `CheckinPage` (`/checkin/:id`, только админ): счётчик отмеченных, форма walk-in, список основы с кнопками «Отметить», список очереди, кнопка «Закрыть чекин» с подтверждением и итогом (сброшено/добрано).
- API-слой `api/checkin.ts`.

## Почему так

- **Самоотметку жёстко ограничил окном и основой** (T4.1), а walk-in и close-checkin — админские, без временных гейтов: админ проводит турнир и действует по факту (решение «гибрид»).
- **Walk-in ищет существующего игрока по телефону** — если человек уже регистрировался раньше, не плодим второй аккаунт; используем его. Если активная запись уже есть — вернётся 409 (уже записан), это ок.
- **Добор из очереди при закрытии — «первому, независимо от чекина»** — ровно как решено. Реализовано циклом `promoteFirstWaitlisted` до заполнения 32.
- **Экран чекина сделал на React (`/checkin/:id`), а не в EasyAdmin** — админ на турнире работает с телефона, нужен mobile-first интерфейс с крупными кнопками «Отметить», чего серверная CRUD-таблица не даёт.
- **`^/api/admin` в access_control** — дублирует `#[IsGranted]` на контроллере: если кто-то забудет атрибут на новом методе, firewall всё равно прикроет.

## Как проверить

Проверено вживую (curl, турнир id=1): walk-in заводит игрока и сразу отмечает (`checkedIn`); тот же телефон в формате `8...` → 409 (не дублируется); `GET /roster` без админ-сессии → 401; `close-checkin` вернул `{dropped:0, promoted:0}` и статус `checkin`. Dev-данные после проверки прибраны.

Через браузер: залогинься админом (`79991234567`/`secret1`) → «Турниры» → «Экран чекина (админ)» на карточке → форма walk-in, списки, «Закрыть чекин».

Самоотметку игрока и обработку очереди на 34 игроках покрывают тесты: `make test` → `OK (22 tests)`.
