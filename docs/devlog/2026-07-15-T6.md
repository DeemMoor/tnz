# T6 — Ход турнира (2026-07-15)

## Что сделал

**Backend:**
- `AdvanceService::recordWinner(match, winner, byAdmin)`:
  - валидация: победитель — один из двух игроков матча (иначе 422);
  - идемпотентность: тот же победитель повторно → no-op;
  - защита: сыгранный матч обычный игрок переписать не может (409); админ — может (откат старого результата + новая запись);
  - продвижение победителя в следующий тур по формуле `(round r, slot s) → (round r+1, slot s>>1)`, player1/player2 по чётности slot;
  - если следующего тура нет — это финал стола → победитель `isChampion = true`;
  - первый результат в `drawn`-турнире → статус `in_progress`; когда финалы всех столов сыграны → `finished`.
- `MatchController` `POST /api/matches/{id}/winner` (ROLE_USER): право отметить — админ ИЛИ участник этого матча; победитель — из двух игроков; иначе 403/422.
- Тесты `AdvanceServiceTest` (+4, всего **31**): один стол → чемпион + finished; два стола (18 игроков) → finished только когда оба финала сыграны, чемпионов двое; победитель обязан быть участником; игрок не переписывает, админ переотмечает (старый убран из финала, новый поставлен). Все зелёные.

## Почему так

- **Формула связи туров — единая с `DrawService`** (там она пробрасывала байи): `slot >> 1` в следующий тур, чётный slot → player1, нечётный → player2. Продвижение и жеребьёвка используют одну и ту же геометрию сетки — нет рассинхрона.
- **Чемпион = победитель матча, у которого нет следующего тура.** Не завожу отдельного поля «финал»: финал определяется топологией (нет матча round+1). Это устойчиво к разному числу туров на столах (стол 2 с 4 игроками имеет 2 тура, стол 1 — 4).
- **Завершение считаю по «финалам всех столов»** (матч с максимальным round каждого стола, slot 0). Пока хоть один финал не done — турнир не finished.
- **Переотметку разрешил только админу** и сделал с откатом: убираю старого победителя из слота следующего матча (и снимаю чемпионство, если это был финал). Если ниже по сетке уже сыграли — глубокий каскад не разматываю (редкий край, админ поправит вручную). Записал как ограничение.
- **Право отметить — админ или участник** проверяю в контроллере (не ролью): это не роль, а принадлежность к конкретному матчу.

## Как проверить

Проверено вживую (curl, турнир id=1, 4 walk-in → draw): админ `POST /api/matches/8/winner {winnerId:9}` → 200, статус `in_progress`, победитель проброшен в финал (матч 10 `player1=9`); невалидный `winnerId` → 422; обычный не-участник → 403. Dev прибран.

Тесты: `make test` → `OK (31 tests)`.
